# 结算功能接口补充文档

## 基础信息

**API 根地址**: `https://oopnpjictkfv.sealoshzh.site/api/v1`

**统一响应格式**:
```json
// 成功响应
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}

// 失败响应
{
  "code": 400,
  "message": "错误信息",
  "data": null
}
```

**认证方式**: JWT Token，在请求头中携带：
```
Authorization: Bearer <token>
```

## 新增/修改接口说明

### 一、创建订单接口（已存在，建议确认以下字段）

**路径**: `/order/create`

**方法**: `POST`

**认证**: 需要

**请求体**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| items | array | 是 | 订单项列表 |
| remark | string | 否 | 备注（用户输入的订单备注，如：少辣、不要香菜等） |

**items 数组结构**:
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| foodId | int | 是 | 菜品ID |
| name | string | 是 | 菜品名称 |
| price | float | 是 | 单价 |
| emoji | string | 否 | 表情符号 |
| quantity | int | 是 | 数量（必须传递） |
| spec | string | 否 | 规格（如果菜品有规格） |

**请求示例**:
```json
{
  "items": [
    {
      "foodId": 1,
      "name": "招牌牛肉锅",
      "price": 68,
      "emoji": "🍲",
      "quantity": 2
    },
    {
      "foodId": 3,
      "name": "宫保鸡丁",
      "price": 32,
      "emoji": "🍛",
      "quantity": 1
    }
  ],
  "remark": "少辣，不要香菜"
}
```

**成功返回**:
```json
{
  "code": 200,
  "message": "下单成功",
  "data": {
    "orderId": "ORD20240605120000",
    "orderNo": "ORD20240605120000",
    "items": [
      {
        "foodId": 1,
        "name": "招牌牛肉锅",
        "price": 68,
        "emoji": "🍲",
        "quantity": 2,
        "subtotal": 136
      },
      {
        "foodId": 3,
        "name": "宫保鸡丁",
        "price": 32,
        "emoji": "🍛",
        "quantity": 1,
        "subtotal": 32
      }
    ],
    "totalPrice": 168,
    "originalPrice": 168,
    "discount": 20,
    "actualPrice": 148,
    "remark": "少辣，不要香菜",
    "status": "pending",
    "statusText": "待接单",
    "createTime": "2024-06-05 12:00:00"
  }
}
```

**关键说明**:
1. **quantity 字段必须传递**：前端结算页面会传入购物车中每个菜品的数量
2. **remark 备注字段**：用户可以在结算页面填写备注信息（少辣、不要香菜等）
3. **优惠计算**：后端需要根据 totalPrice 计算优惠（满100减20，满200减50）
4. **返回实际付款金额**：actualPrice = originalPrice - discount

### 四、再来一单接口（快速复购）

**路径**: `/order/reorder/{order_id}`

**方法**: `POST`

**认证**: 需要

**说明**: 将历史订单中的菜品快速添加到当前购物车

**路径参数**:
| 参数 | 类型 | 说明 |
|------|------|------|
| order_id | string | 历史订单ID |

**成功返回**:
```json
{
  "code": 200,
  "message": "已添加到购物车",
  "data": {
    "items": [
      {
        "foodId": 1,
        "name": "招牌牛肉锅",
        "price": 68,
        "emoji": "🍲",
        "quantity": 2
      }
    ],
    "totalCount": 2,
    "totalPrice": 136
  }
}
```

**错误返回**:
```json
{
  "code": 404,
  "message": "订单不存在或该订单无可复购商品",
  "data": null
}
```

## 优惠规则说明

### 前端优惠计算逻辑

前端会根据总价自动计算优惠：

| 订单金额 | 优惠 | 实付金额 |
|----------|------|----------|
| < 100元 | 0元 | 原价 |
| 100-199元 | 20元 | 原价 - 20 |
| >= 200元 | 50元 | 原价 - 50 |

### 后端优惠计算（建议）

后端也需要根据同样的规则计算优惠金额：

```javascript
function calculateDiscount(totalPrice) {
  if (totalPrice >= 200) {
    return 50;
  } else if (totalPrice >= 100) {
    return 20;
  }
  return 0;
}
```

## 结算页面流程说明

### 1. 进入结算页面
- 用户点击购物车底部的"结算"按钮
- 前端跳转到结算页面 `/pages/checkout/checkout`
- 结算页面从本地购物车（Pinia store）获取商品列表
- 展示商品明细、厨师信息、备注输入框

### 2. 修改商品数量
- 用户可以在结算页面直接修改商品数量
- 前端实时更新总价和优惠金额

### 3. 填写备注
- 用户在备注输入框填写特殊要求（少辣、不要香菜等）
- 备注信息将随订单一起提交

### 4. 提交订单
- 用户点击"提交订单"按钮
- 前端调用 `/order/create` 接口
- 成功后调用 `/cart/clear` 清空购物车
- 跳转到订单页面

### 5. 后端处理
1. 接收订单创建请求
2. 验证用户登录状态
3. 计算优惠金额
4. 创建订单记录
5. 清空购物车
6. 返回订单信息

## 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 成功 |
| 400 | 请求参数错误（如缺少必填字段） |
| 401 | 未登录或Token失效 |
| 404 | 资源不存在（如订单不存在） |
| 422 | 验证失败 |
| 500 | 服务器内部错误 |
