# 接口变更说明

## 变更概述

| 项目 | 内容 |
|------|------|
| **变更类型** | 接口方法变更 |
| **变更原因** | 用户评分需覆盖之前的评分，而非新增记录 |
| **影响范围** | `/review/submit` 接口 |
| **变更时间** | 2026-07-01 |

---

## 接口详情

### 原接口定义

| 属性 | 值 |
|------|-----|
| **请求方法** | `POST` |
| **接口路径** | `/api/v1/review/submit` |
| **功能描述** | 新增用户评价记录 |

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `rating` | Integer | 是 | 评分（1-5） |
| `suggestion` | String | 否 | 建议内容 |

**响应示例**：

```json
{
  "code": 200,
  "msg": "评价提交成功",
  "data": {
    "id": 1,
    "rating": 5,
    "suggestion": "很好用",
    "createdAt": "2026-07-01 10:00:00"
  }
}
```

---

### 新接口定义

| 属性 | 值 |
|------|-----|
| **请求方法** | `PUT` |
| **接口路径** | `/api/v1/review/submit` |
| **功能描述** | **更新/覆盖**用户评价记录（若用户已有评价则更新，无则新增） |

**请求参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `rating` | Integer | 是 | 评分（1-5） |
| `suggestion` | String | 否 | 建议内容 |

**响应示例**：

```json
{
  "code": 200,
  "msg": "评价更新成功",
  "data": {
    "id": 1,
    "rating": 4,
    "suggestion": "更新后的建议",
    "updatedAt": "2026-07-01 11:00:00"
  }
}
```

---

## 后端处理逻辑要求

```
用户提交评价 → 检查用户是否已有评价记录
              ↓
         ┌────┴────┐
         ↓         ↓
      已有记录    无记录
         ↓         ↓
   更新现有记录   新增记录
         ↓         ↓
   重新计算积分   计算积分
         ↓         ↓
      返回成功    返回成功
```

### 积分计算规则

| 评分 | 积分变化 |
|------|----------|
| 1星 | +1分 |
| 2星 | +2分 |
| 3星 | +3分 |
| 4星 | +4分 |
| 5星 | +5分 |

> **注意**：使用 PUT 方式时，积分计算需基于最新评分重新计算，而非累加。

## 调用示例

### 原请求（POST - 新增）

```bash
curl -X POST https://example.com/api/v1/review/submit \
  -H "Authorization: Bearer xxx" \
  -H "Content-Type: application/json" \
  -d '{"rating": 5, "suggestion": "很好"}'
```

### 新请求（PUT - 更新/覆盖）

```bash
curl -X PUT https://example.com/api/v1/review/submit \
  -H "Authorization: Bearer xxx" \
  -H "Content-Type: application/json" \
  -d '{"rating": 4, "suggestion": "还不错"}'
```

---

## 注意事项

1. **幂等性**：PUT 请求应具有幂等性，多次调用相同参数应返回相同结果
2. **Token验证**：需验证请求头中的 Authorization Token
3. **积分更新**：每次更新评分后需重新计算用户积分（`points` 字段）
4. **响应时间**：建议接口响应时间 ≤ 500ms
