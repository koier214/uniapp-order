# 项目功能总结文档

## 一、项目概览

### 技术栈
- **框架**：uni-app（跨平台应用开发）
- **核心**：Vue3
- **动画库**：lottie-web
- **数据存储**：本地存储（购物车）
- **API管理**：统一封装在 `src/utils/request.js`

### 项目配置
- **API地址**：可配置 `src/utils/request.js` 中的 `BASE_URL`
  - 当前：公网调试地址 `https://qhhxncfdtcyd.sealoshzh.site/api/v1`
  - 备选：线上地址 `https://oopnpjictkfv.sealoshzh.site/api/v1`
- **页面路由**：`src/pages.json`
- **组件库**：使用uni-app内置组件 + 自定义组件

## 四、需要完善的问题

### 🔴 高优先级问题
1. **购物车未对接后端API**
   - 当前：使用本地存储（`store/cart.js`）
   - 问题：多设备不同步，没有持久化到后端
   - API已定义在 `api/cart.js` 但未使用

2. **结算页厨师信息未对接API**
   - 当前：硬编码显示（"金牌厨师 - 王师傅"）
   - 问题：无法展示真实厨师信息
   - API已定义在 `api/chef.js`

3. **用户信息未实时获取**
   - 当前：`menu.vue` 从本地存储读取
   - 问题：后端更新了信息，前端无法刷新
   - 建议：页面onShow时调用 `getUserInfo()` 接口

### 🟡 中优先级问题
4. **分类菜单硬编码**
   - 当前：`menu.vue` 分类写死
   - 建议：调用 `getCategories()` API动态获取

5. **订单详情页缺失或不完善**
   - 当前：只有订单列表，没有详情页

6. **菜品规格选择功能缺失**
   - API返回中有 `specs` 字段，但页面未使用

### 🟢 低优先级问题
7. **缺少用户等级和积分展示**
   - API返回中有 `points`、`level`、`days` 字段

8. **缺少支付功能**
   - 创建订单后需要支付流程

9. **缺少地址管理功能**
   - 外卖场景需要配送地址

10. **缺少搜索功能**
    - 无法搜索菜品

11. **缺少收藏功能**
    - 无法收藏喜欢的菜品

## 六、开发建议（从开发者角度）

### 1. 代码结构优化
- **统一状态管理**：建议使用 Pinia/Vuex 替代现在简单的 cart.js store
- **API调用抽象**：将重复的API调用逻辑封装为 composable
- **错误处理统一**：现在各页面的错误处理方式不统一

### 2. 购物车对接建议
```javascript
// 在 store/cart.js 中新增对接逻辑
import { getCartList, addToCart, updateCartItem, removeCartItem, clearCart } from '../api/cart.js'

// 改造现有方法，优先调用API，失败时降级为本地存储
```

### 3. 页面数据更新策略
```javascript
// 在 onShow 中主动刷新数据
async onShow() {
  // 刷新用户信息
  await this.loadUserInfo()
  // 刷新购物车（从后端）
  await this.loadCartFromServer()
}
```

### 4. 调试建议
- 已在 `checkout.vue` 中添加了调试日志
- 建议所有API调用都添加 `console.log` 便于调试
- 关注 Network 面板，确认请求是否正常发送

### 5. 测试建议
- 测试登录注册流程
- 测试添加购物车 -> 结算 -> 下单 -> 查看订单 完整流程
- 测试购物车拖拽功能
- 测试各页面空状态展示（API失败或无数据时）
